📝 Padrão de Commits (Conventional Commits)

Usamos o padrão Conventional Commits
.

Formato:

<tipo>(escopo): descrição breve

🔑 Tipos principais:

feat → nova funcionalidade

fix → correção de bug

docs → apenas documentação

style → ajustes de formatação (espaços, ponto e vírgula, etc)

refactor → refatoração sem mudar funcionalidade

test → inclusão ou ajustes em testes

chore → tarefas de manutenção (configs, dependências)

✅ Exemplos:
feat(backend): criar endpoint para agendamento
fix(frontend): corrigir erro no formulário de login
docs: atualizar README com instruções de instalação
chore: atualizar dependências do Docker
